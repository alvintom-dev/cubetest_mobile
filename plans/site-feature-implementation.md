# Site Feature — Clean Architecture Implementation Plan

## Context

`cubetest_mobile` is a fresh Flutter app — only `lib/main.dart` has starter counter code; no `core/`, `config/`, `shared/`, `services/`, or `features/` exist, and `pubspec.yaml` has only Flutter boilerplate deps. `specs/site.md` defines a Site CRUD feature with soft-delete/archive plus a `testSets: int` count that conceptually originates from the (out-of-scope) test_set feature.

CLAUDE.md mandates strict Clean Architecture, specific packages (dartz, get_it, injectable, go_router, bloc, equatable, json_annotation, mocktail, bloc_test), and a fixed folder layout. The Site feature cannot compile without shared infrastructure, so this plan also scaffolds the minimum `core/`, `config/`, plus empty `shared/` and `services/` to match the full top-level CLAUDE.md layout.

### Decisions confirmed with user
1. **Shared scaffold**: create the full top-level CLAUDE.md layout (`core/`, `config/`, `shared/`, `services/`), even though Site only actively uses `core/` + `config/` right now.
2. **Storage**: **Local only via sqflite** — no remote datasource. Per CLAUDE.md "For read-only features with no local storage, local_datasource may be omitted" → this is the inverse (local-only, no remote), which is explicitly approved here.
3. **`Site.testSets` count**: hardcode `0` in repository with a `// TODO: wire to test_set local datasource when test_set feature is implemented`.
4. **Cascade delete of test sets/tests**: no-op in Site repository with a `// TODO: cascade-delete test sets & tests when those features exist`.

## Phase 3 — Domain layer

### Entity `Site`
```dart
enum SiteStatus { open, close }

class Site extends Equatable {
  const Site({
    required this.id,
    required this.name,
    this.location,
    this.description,
    this.status = SiteStatus.open,
    required this.createdAt,
    required this.updatedAt,
    this.deletedAt,
    this.isDeleted = false,
    this.testSets = 0,
  });
  final String id;
  final String name;
  final String? location;
  final String? description;
  final SiteStatus status;
  final DateTime createdAt;
  final DateTime updatedAt;
  final DateTime? deletedAt;
  final bool isDeleted;
  final int testSets;

  @override
  List<Object?> get props => [id, name, location, description, status, createdAt, updatedAt, deletedAt, isDeleted, testSets];
}
```

### Repository interface `SiteRepository`
- `Future<Either<Failure, Site>> createSite({required CreateSiteUsecaseParam param})`
- `Future<Either<Failure, List<Site>>> getSites()` — excludes deleted
- `Future<Either<Failure, List<Site>>> getArchivedSites()` — only deleted
- `Future<Either<Failure, Site>> getSite({required String id})`
- `Future<Either<Failure, Site>> updateSite({required UpdateSiteUsecaseParam param})`
- `Future<Either<Failure, bool>> deleteSite({required DeleteSiteUsecaseParam param})`

### Usecases (one per file, all `@injectable`, extend `UseCase<Either<Failure,T>, P>`)

- **`CreateSiteUsecase`** / `CreateSiteUsecaseParam(name, location?, description?, status)` — validates `name.trim().isNotEmpty` else `ValidationFailure`.
- **`GetSitesUsecase`** (no params via `NoParamsUseCase<Either<Failure, List<Site>>>`).
- **`GetArchivedSitesUsecase`** (no params).
- **`GetSiteUsecase`** / `GetSiteUsecaseParam(id)` — validates non-empty id.
- **`UpdateSiteUsecase`** / `UpdateSiteUsecaseParam(id, name, location?, description?, status)` — validates id + non-empty name.
- **`DeleteSiteUsecase`** / `DeleteSiteUsecaseParam(id, isPermanentDelete)` — validates id.

Each returns `Left(InternalAppError('Params cannot be null'))` when params null, per CLAUDE.md §USECASE RULE example.

## Phase 4 — Data layer

### `SiteModel` (DTO in `data/models/site_model.dart`)
`@JsonSerializable()` with fields matching the entity (status stored as String `"open"|"close"`, dates as ISO8601 strings, testSets NOT persisted — always 0 from DB row). `fromJson` / `toJson` / `fromMap` / `toMap` (sqflite maps) + `toEntity()` + `fromEntity(Site)`.

### `SiteLocalDatasource` (`@injectable`, `data/datasource/site_local_datasource.dart`)
Wraps a sqflite `Database`. Exposes:
- `Future<SiteModel> create(SiteModel)`
- `Future<List<SiteModel>> getAll({bool includeDeleted = false})`
- `Future<List<SiteModel>> getArchived()`
- `Future<SiteModel?> getById(String id)`
- `Future<SiteModel> update(SiteModel)`
- `Future<void> softDelete(String id)`  — sets `isDeleted=1`, `deletedAt=now`
- `Future<void> hardDelete(String id)`

DB opening via `path_provider` + `path.join`. Schema:
```sql
CREATE TABLE sites (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  location TEXT,
  description TEXT,
  status TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  deleted_at TEXT,
  is_deleted INTEGER NOT NULL DEFAULT 0
);
```
Throws plain `Exception` on DB errors; the repository maps to `Failure`.

### `SiteRepositoryImpl` (`@Injectable(as: SiteRepository)`, `data/repository/site_repository_impl.dart`)
Constructor-injects `SiteLocalDatasource` + `Uuid` (for id gen). All methods:
- try/catch → `CacheFailure(e.toString())` on any exception.
- `createSite`: generate id (uuid v4), set `createdAt=updatedAt=now`, status default `open`, `isDeleted=false`, `testSets=0`.
- `updateSite`: fetch existing or return `CacheFailure('Site not found')`; overwrite mutable fields, bump `updatedAt`.
- `deleteSite`: if `isPermanentDelete` → `hardDelete`, else `softDelete`.
  - `// TODO: cascade-delete related test sets & tests when those features are implemented.`
- Entity mapping: `model.toEntity().copyWith(testSets: 0)` — with `// TODO: wire to test_set local datasource (totalTestSetsBySiteId) when test_set feature is implemented.`

(Entity will need a `copyWith` — add it.)

## Phase 5 — Presentation layer

### State flow per CLAUDE.md §LOADING STATE RULE

#### `SitesCubit` (list)
States (sealed via base `SitesState extends Equatable`):
- `SitesInitial`, `SitesLoading`, `SitesLoadedData(List<Site>)`, `SitesLoadedNoData`, `SitesError(String)`
Actions: `fetch()` — emits Loading then delegates to `GetSitesUsecase`; empty list → `SitesLoadedNoData`.

#### `ArchivedSitesCubit` (same pattern for archive page — backed by `GetArchivedSitesUsecase`). Co-located with SitesCubit or its own folder; keep one cubit per directory per CLAUDE.md — use its own folder `sites_archive_cubit/`.

#### `SiteDetailCubit`
States: `SiteDetailInitial`, `SiteDetailLoading`, `SiteDetailLoaded(Site)`, `SiteDetailError(String)`.
Action: `load(id)`.

#### `SiteFormCubit`
States: `SiteFormInitial`, `SiteFormSubmitting`, `SiteFormSuccess(Site)`, `SiteFormValidationError(String)`, `SiteFormError(String)`.
Actions: `submitCreate(param)`, `submitUpdate(param)`. Maps `ValidationFailure → SiteFormValidationError`, others → `SiteFormError`.

All cubits resolved via `getIt` (per CLAUDE.md §DI). Pages wrap with `BlocProvider(create: (_) => getIt<XCubit>())`.

### Pages
- `SitesPage`: AppBar with "Archive" action + FAB to create. `BlocBuilder` renders Loading/Empty/Error/List.
- `SitesArchivePage`: lists soft-deleted sites; swipe-to-permanent-delete.
- `SiteDetailPage`: shows Site fields + `testSets` count + edit/delete buttons. "Associated test sets" section renders placeholder `Text('0 test sets')` with a comment stating it'll be populated once test_set feature exists.
- `SiteFormPage`: used for both create and edit (router wires `id?` via path param). Uses `SiteForm` widget.

### Widgets
- `SiteListItem`: shows name, `testSets` count, `SiteStatusChip`.
- `SiteStatusChip`: colored chip keyed off `SiteStatus`.
- `SiteForm`: text fields + dropdown for status; `validator` on name for non-empty; `onSubmit` returns a param object.

### Navigation wiring (in `core/route/router.dart`)
```dart
GoRoute(path: AppRoutes.sitesPath, name: AppRoutes.sites, builder: (_, __) => const SitesPage(), routes: [
  GoRoute(path: 'create', name: AppRoutes.siteCreate, builder: ...),
  GoRoute(path: 'archive', name: AppRoutes.sitesArchive, builder: ...),
  GoRoute(path: ':id', name: AppRoutes.siteDetail, builder: ...),
  GoRoute(path: ':id/edit', name: AppRoutes.siteEdit, builder: ...),
]),
```
